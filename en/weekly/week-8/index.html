Codex Usage

GitHub - Dailin521/codex-provider-sync: Synchronize Codex session provider metadata across rollout files and SQLite state. · GitHub — if you use both official subscriptions and third-party APIs, Codex history may fail to display because the provider identifiers differ. This tool synchronizes them so the history shows up correctly.

If you use the official subscription, you may need TUN mode to avoid reconnecting issues. If you do not want to enable TUN mode, create a .env file under ~/.codex and add proxy settings like:

HTTP_PROXY="http://127.0.0.1:your-proxy-port" # clash seems to use 7897 by default
HTTPS_PROXY="http://127.0.0.1:your-proxy-port"
NO_PROXY="localhost,127.0.0.1"
