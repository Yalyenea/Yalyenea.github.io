Codex 使用

GitHub - Dailin521/codex-provider-sync: Synchronize Codex session provider metadata across rollout files and SQLite state. · GitHub如果你同时使用官方订阅以及第三方 API 的话。那么 Codex 的历史对话可能会不显示，原因就是因为标识符好像不一样。这个工具可以进行统一同步，然后就能完美看到历史对话了。

然后如果你使用官方订阅的话，需要开 tun 模式才不会出现 reconnecting。但是如果你又不想开 tun 模式呢，你可以在 ~/.codex 目录下新建一个 .env 文件，加入以下的代理配置：

HTTP_PROXY="http://127.0.0.1:your-proxy-port" # clash好像默认 7897 
HTTPS_PROXY="http://127.0.0.1:your-proxy-port"
NO_PROXY="localhost,127.0.0.1"
